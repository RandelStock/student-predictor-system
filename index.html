01

System Purpose

The system predicts EE licensure outcomes for students and provides a comprehensive analytics layer for faculty.

🎯

Pass / Fail Prediction

Binary classification with probability scores for both pass and fail outcomes.

📊

Predicted Rating

Two regression variants (A and B) providing numeric rating estimates.

🔍

Reliability Score

Measures the reliability of questionnaire responses from the student.

🤖

AI Recommendations

Groq-powered personalized feedback per section of the assessment.

📈

Professor Analytics

Institutional trends, model metrics, and live monitoring dashboard.

03

High-Level Architecture

🖥️

Frontend frontend/

Role-based app shell — landing, login, student page, professor page. Calls backend via REST using API_BASE_URL. Stores JWT token and role in localStorage.

⚙️

Backend backend/

Loads trained model bundle ree_survey_model.pkl. Exposes prediction, recommendation, analytics, admin, and defense endpoints. Uses FACULTY_CODE to gate professor registration. Institutional analytics source: DATA_UPCOMING.xlsx (333 rows, 2022–2025).

🗄️

Database

Tables: users, prediction_attempts, recommendations_cache. Managed via backend/database.py.

10

Machine Learning Pipeline

Implemented in backend/train_model.py.

🌲

Classification

RandomForestClassifier — predicts Pass / Fail. Train on DATA_MODEL.xlsx (60, with survey). Test on DATA_TEST.xlsx (21, 2025 held-out).

📐

Regression A

RandomForestRegressor using basic features (EE, MATH, ESAS, GWA). Train on DATA_MODEL (60) + DATA_SYSTEM 2022–2024 slice (~250) = ~310. Test on DATA_TEST (21).

📐

Regression B

RandomForestRegressor excluding subject scores — GWA + survey only. Train on DATA_MODEL (60), test on DATA_TEST (21).

Model Bundle Exports

  • Trained models (classifier + 2 regressors)
  • Feature lists: FEATURES_ALL, FEATURES_BASIC, FEATURES_NOSUB
  • Encoders for categorical variables
  • Evaluation metrics

Runtime Inference

The /predict endpoint calls build_feature_vector() to reconstruct feature vectors from the incoming request before passing them through each model.

Data Files and Roles

  • DATA_MODEL.xlsx — 60 rows, 98 cols (2022–2025). With survey. Primary training source.
  • DATA_SYSTEM.xlsx — 333 rows, 8 cols (2022–2025). No survey. Used only for Regression A (2022–2024 slice) to avoid leakage.
  • DATA_TEST.xlsx — 21 rows (2025 Apr+Aug). With survey. Held-out evaluation set.
  • DATA_UPCOMING.xlsx — 333 rows (2022–2025). No survey. Single source of truth for professor dashboard analytics.

Data Flow Summary

📥

Input Sources

Student form entries, historical Excel datasets for institutional analytics, DB attempts for live monitoring.

⚙️

Processing

Feature vector assembly → ML inference (classifier + 2 regressors) → reliability computation → optional Groq AI text generation.

💾

Storage

Prediction results + full input snapshot in DB. AI recommendation text cached per user + attempt + section.

📤

Output

Student-focused prediction result card. Professor-focused aggregate dashboards and trend reports.

14

Important Operational Notes

Missing DATABASE_URL — Prediction can still run, but persistence, history, and live monitoring become limited.
Missing GROQ_API_KEY — AI recommendation and AI trend summary endpoints return fallback text rather than generated responses.
API URL — Frontend uses the fixed production URL in apiBase.js. Change this for local development.
